# 系统设置新功能 - 快速上手指南

## 🎯 功能概览

本次更新在**系统设置**页面新增了三个核心管理模块:

```
┌─────────────────────────────────────┐
│       系统设置页面新增模块           │
├─────────────────────────────────────┤
│ 1️⃣ 自定义字段管理                    │
│    - 为职工信息添加动态扩展字段       │
│    - 支持文本/数字/日期/下拉等类型    │
│    - 可按科室配置不同字段             │
├─────────────────────────────────────┤
│ 2️⃣ 工资项目管理                      │
│    - 配置工资核算的各项收入/扣款项目   │
│    - 支持固定项/浮动项/扣款项         │
│    - 可设置计税规则和计算公式         │
├─────────────────────────────────────┤
│ 3️⃣ 绩效管理分类                      │
│    - 建立绩效考核的分类体系           │
│    - 配置考核指标和评分标准           │
│    - 支持多维度权重分配               │
└─────────────────────────────────────┘
```

## 📋 使用教程

### 1️⃣ 自定义字段管理

#### 场景: 为医生添加"专业特长"字段

**操作步骤:**

1. 点击 **"+ 新增字段"** 按钮
2. 填写表单:
   ```
   字段名称: 专业特长
   字段编码: specialty
   字段类型: 文本
   适用科室: 留空(适用于所有科室)
   是否必填: 否
   排序: 1
   状态: 启用
   ```
3. 点击 **"保存"**

**效果:** 
- 在职工信息管理页面,可以为每位医生录入其专业特长
- 支持按科室筛选显示不同的自定义字段

#### 高级用法: 创建下拉选择字段

**示例: 为护士添加"护理等级"字段**

```
字段名称: 护理等级
字段编码: nursing_level
字段类型: 下拉选择
选项: N0,N1,N2,N3,N4
适用科室: 护理部
是否必填: 是
```

---

### 2️⃣ 工资项目管理

#### 场景: 添加"交通补贴"工资项目

**操作步骤:**

1. 点击 **"+ 新增项目"** 按钮
2. 填写表单:
   ```
   项目名称: 交通补贴
   项目编码: transport_allowance
   项目类型: 固定项
   计算方式: 固定值
   是否计税: 是
   是否显示: 是
   排序: 10
   描述: 职工月度交通补贴
   ```
3. 点击 **"保存"**

**效果:**
- 该工资项目将出现在工资核算系统中
- 可以为不同科室/职称设置不同的默认金额

#### 高级用法: 设置项目默认值

**为不同职称设置不同的基本工资:**

```
项目: 基本工资
- 正高职称: 8000元
- 副高职称: 6500元
- 中级职称: 5000元
- 初级职称: 3500元
```

通过API设置:
```bash
POST /api/salary-items/1/defaults
{
  "department": null,
  "title_level": "正高",
  "default_value": 8000
}
```

### 3️⃣ 绩效管理分类

#### 场景: 创建"工作业绩"考核分类

**操作步骤:**

1. 点击 **"+ 新增分类"** 按钮
2. 填写表单:
   ```
   分类名称: 工作业绩
   分类编码: work_performance
   权重: 40
   描述: 工作完成情况和质量评估
   排序: 1
   ```
3. 点击 **"保存"**

#### 添加考核指标

1. 在"工作业绩"行点击 **"+ 指标"** 按钮
2. 填写指标信息:
   ```
   指标名称: 门诊量
   指标编码: outpatient_count
   评分方式: 百分制
   满分: 100
   权重: 30
   描述: 月度门诊接诊患者数量
   ```
3. 点击 **"保存"**

**效果:**
- 建立完整的绩效考核体系
- 支持多维度、多指标的量化考核
- 可为不同岗位配置不同的考核方案

#### 典型绩效体系示例

```
│
├─ 服务质量 (30%)
│  ├─ 患者满意度 (40%)
```

## 💡 实用技巧

### 技巧1: 批量导入自定义字段

如果需要为多个科室配置相同的字段,可以:

1. 先创建一个通用字段(适用科室留空)
2. 再为特殊科室创建专用字段

### 技巧2: 工资项目分组管理

建议按以下顺序排列工资项目:

```
1-10:  基本收入项目 (基本工资、岗位津贴等)
11-20: 补贴项目 (医疗补贴、住房补贴等)
21-30: 浮动收入 (绩效奖金、加班费等)
31-40: 扣款项目 (社保、公积金、个税等)
```

### 技巧3: 绩效指标量化

尽量使用可量化的指标:

✅ **好的指标:**
- 门诊量 ≥ 200人次/月 → 100分
- 患者满意度 ≥ 95% → 100分
- 论文发表 ≥ 2篇/年 → 100分

❌ **不好的指标:**
- 工作态度好
- 服务意识强

### 技巧4: 定期审查和优化

建议每季度审查一次:

- [ ] 检查自定义字段的使用情况,删除不再使用的字段
- [ ] 调整工资项目的金额标准
- [ ] 优化绩效指标的权重分配
- [ ] 根据实际数据调整考核标准

## 🔧 API调用示例

```bash
curl -X POST http://127.0.0.1:5000/api/custom-fields \
  -H "Content-Type: application/json" \
  -b "session=YOUR_SESSION_COOKIE" \
  -d '{
    "field_name": "执业证书编号",
    "field_code": "license_number",
    "field_type": "text",

    "sort_order": 1,
```

## ❓ 常见问题

### Q1: 如何为不同科室显示不同的自定义字段?

**A:** 创建字段时,在"适用科室"字段填写科室名称。例如:
- 留空: 所有科室都显示
- "内科": 仅内科显示
- "外科": 仅外科显示

### Q2: 工资项目创建后如何设置默认金额?

**A:** 目前需要通过API设置默认值:

```bash
POST /api/salary-items/{item_id}/defaults
{
  "department": "内科",
  "title_level": "正高",
  "default_value": 8000
}
```

后续版本将在前端界面添加默认值设置功能。

### Q3: 绩效指标的权重总和必须是100%吗?

**A:** 不是强制要求,但建议同一分类下的指标权重总和为100%,这样便于计算最终得分。

### Q4: 删除工资项目会影响历史工资数据吗?

**A:** 不会。删除操作只会删除项目定义,不会影响已经生成的工资记录。

### Q5: 如何禁用某个字段而不删除?

**A:** 编辑字段,将"状态"设置为"禁用"。这样该字段不会在新建/编辑职工时显示,但历史数据仍保留。
