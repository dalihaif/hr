## 💰 绩效工资计算公式

### 公式定义

```
最终绩效 = 基准绩效 × 岗位系数 × 分数系数

其中:
- 基准绩效: 从科室系数表获取(如:3000元)
- 岗位系数: 根据科室和岗位类型确定(如:1.2)
- 分数系数 = 平均分 / 100
  - 平均分 = (质量分 + 服务分 + 技能分 + 贡献分) / 4
```

### 计算示例

**某内科医师张三的绩效:**

| 项目 | 数值 |
|------|------|
| 基准绩效 | 3,000元 |
| 岗位系数 | 1.2 (内科医师) |
| 工作质量分 | 95 |
| 服务态度分 | 90 |
| 专业技能分 | 92 |
| 贡献度分 | 88 |
| **平均分** | **(95+90+92+88)/4 = 91.25** |
| **分数系数** | **91.25/100 = 0.9125** |
| **最终绩效** | **3000 × 1.2 × 0.9125 = 3,285元** |

## 🗄️ 数据库设计(7张表)

### 1. department_coefficients - 科室岗位系数表
存储各科室各岗位的系数和基准绩效

### 2. performance_periods - 考核周期表
管理考核周期的创建、发布、审核状态

### 3. performance_dept_reports - 科室绩效申报表
科室主任填报的本科室绩效汇总

### 4. performance_emp_details - 职工绩效明细表
每个职工的详细评分和绩效金额

### 5. performance_audit_logs - 绩效审核日志表
记录所有审核操作,可追溯

### 6. performance_payments - 绩效发放记录表
财务发放记录和银行转账信息

### 7. performance_statistics - 绩效统计表
年度/月度统计汇总

## 🔧 API接口(18个)

### 科室系数管理(4个)
- `GET /api/performance/coefficients` - 获取系数列表
- `POST /api/performance/coefficients` - 新增系数
- `PUT /api/performance/coefficients/<id>` - 修改系数
- `DELETE /api/performance/coefficients/<id>` - 停用系数

### 考核周期管理(4个)
- `GET /api/performance/periods` - 获取周期列表
- `POST /api/performance/periods` - 创建周期
- `POST /api/performance/periods/<id>/publish` - 发布周期
- `POST /api/performance/periods/<id>/approve` - 审核周期

### 科室申报管理(6个)
- `GET /api/performance/dept-reports` - 获取申报列表
- `POST /api/performance/dept-reports` - 提交申报
- `PUT /api/performance/dept-reports/<id>` - 更新申报
- `POST /api/performance/dept-reports/<id>/submit` - 提交审核
- `POST /api/performance/dept-reports/<id>/review` - 一级审核
- `POST /api/performance/dept-reports/<id>/final-approve` - 最终审批

### 其他(4个)
- `GET /api/performance/emp-details` - 获取个人明细
- `POST /api/performance/emp-details` - 批量导入
- `POST /api/performance/payments/confirm` - 财务确认
- `GET /api/performance/statistics` - 统计数据
